- **数据层**：`llm_provider.rs` 定义了 `ModelProvider`、`ProviderRegistry` 及 LLM 请求/响应结构（如 `LLMRequest`、`ChatMessage`），内置 DeepSeek、OpenAI 等主流提供商配置。
- **通信层**：`llm_stream.rs` 实现 `StreamingLLMClient`，基于 `reqwest` 处理 SSE 流解析、指数退避重试，并通过 Tauri `Emitter` 向前端推送 `llm-stream-token` 等事件。
- **治理层**：`token_runtime_engine.rs` 负责 Token 使用记录的持久化逻辑（内存态）、多维度（日/月/年）配额检查（`check_quota`）及仪表盘数据聚合（`build_dashboard_snapshot`）。
- **增强层**：`web_search.rs` 提供 Bing/DuckDuckGo 搜索 fallback 机制、结果缓存（`SearchCache`）及网页正文提取，作为 LLM 的外部知识补充工具。
