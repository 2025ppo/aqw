- **核心组件分层**：`DraftCanvas` 负责底层 Canvas 渲染与事件处理，`DraftToolbox` 管理 UI 工具栏状态，`InfiniteCanvas` 与 `FileCanvas` 分别处理项目节点图和文档结构图的 SVG 渲染。
- **渲染架构**：采用主 Canvas 与离屏 Canvas (`offscreenCanvas`) 结合的双缓冲机制，通过 `scheduleRender` 配合 `requestAnimationFrame` 实现按需重绘与视口裁剪优化。
- **数据模型**：定义了 `Stroke`, `Shape`, `Note`, `MiniCanvas` 等结构化接口，并通过 `DraftLayer` 实现内容分层管理，支持可见性与锁定状态控制。
- **交互边界**：通过 `Viewport` 对象统一管理屏幕坐标与世界坐标的转换，确保在缩放和平移状态下绘图、选区及 DOM 元素（如便签）位置的同步一致性。
