- **分层架构**：底层为 `code_chunker`（语言感知分段）与 `tfidf`/`code_graph`（向量化与关系提取），中层为 `perceptual_index`（融合搜索与逻辑画布生成），上层为 `repo_wiki`（AI驱动的知识凝练）与 `experience`（专家行为分析）。
- **数据流向**：原始代码/文档 → `doc_processor`/`code_chunker` 标准化 → `perceptual_index` 构建持久化索引（JSON） → `repo_wiki` 调用 DeepSeek API 生成 Knowledge Cards 与 Wiki 文章。
- **核心接口**：`build_index` 触发全量索引构建，`search` 执行 TF-IDF 与图谱权重的融合检索，`generate_experience_沉淀` 从任务历史中提取记忆、技能与策略维度数据。
- **依赖边界**：通过 `serde` 定义统一数据结构，依赖 `lopdf`/`calamine`/`docx-rs` 处理异构文档，通过 `reqwest` 与外部 LLM 交互完成知识二次加工。
