## Task 2: 修复 focusOnContent 实现自动缩放适配
- 当前 bug：`x = visibleCenterX - centerX * this.view.scale` 中 scale 未重置
- 修复步骤：
  1. 先计算节点包围盒（minX/maxX/minY/maxY + padding）
  2. 获取画布可视区域尺寸（宽 - 437px 左侧对话区）
  3. 计算适配缩放：`scale = min(可视宽/节点宽, 可视高/节点高, 1.0)`，限制 [0.3, 1.5]
  4. 用新 scale 计算 view.x / view.y：`x = visibleCenterX - centerX * scale`
  5. 无节点时保持默认 `{ x: 280, y: 0, scale: 1 }`

## Task 5: Rust 后端 - .xt 配置文件持久化目录数据
- 修改 `ensure_xt_config`：在 `config.json` 默认内容中新增 `canvasDirectory` 字段：
  ```json
  {
    "project": "",
    "version": "0.1.0",
    "files": [],
    "canvasDirectory": {
      "nodes": [],
      "edges": [],
      "updatedAt": ""
    }
  }
  ```
- 新增 Tauri 命令 `save_canvas_directory(projectName, data)`：
  - 定位到 `workspaces/{project}/.xt/config.json`
  - 读取 → 更新 `canvasDirectory` 字段 → 写回
- 新增 Tauri 命令 `load_canvas_directory(projectName)`：
  - 读取 `config.json` 中的 `canvasDirectory`，不存在返回 null

## Task 6: 前端 - 目录数据加载与更新逻辑
- `main.ts` 中新增 `loadProjectCanvas` 重构：
  1. 先调用 `load_canvas_directory` 尝试读取缓存
  2. 如果有缓存且 `updatedAt` 不为空 → 直接 `canvas.setData()`
  3. 无缓存 → 调用 `analyze_project_dependencies`（AI生成）
  4. AI 返回后 → `save_canvas_directory` 持久化
  5. 更新目录卡片上的 `updatedAt` 显示
- 绑定目录卡片的"更新"按钮：
  - 点击 → 强制重新调用 AI → 持久化 → 刷新画布
  - 更新按钮显示 loading 状态（旋转 SVG）

## Task 7: AI 自动更新检测
- 当 AI 执行文件操作（CREATE_FOLDER/CREATE_FILE/WRITE_FILE/DELETE）后
- 检查当前是否为"目录"模式且操作的是当前项目
- 如果是 → 自动触发目录重新生成并持久化
- 更新目录卡片上的时间显示

## Task 8: 结构图优化 - 根目录节点与无关目录过滤
- **问题**：结构模式下根目录文件独立悬浮，且 `node_modules`、`.git`、`.xt`、`dist` 等无关目录全部展示，导致图表杂乱
- **方案**：
  1. 在 `generateStructureCanvas` 中包裹一个根目录节点（以项目名命名），将所有条目作为其子节点
  2. 递归过滤无关目录：`node_modules`, `target`, `dist`, `.git`, `.xt`, `__pycache__`, `build`, `.next`, `.nuxt`, `out`, `.vscode`, `.idea`, `venv`, `.venv`
- **注意**：这个过滤仅作用于结构图展示，不影响文件浏览器（资源管理器仍需显示完整目录）
